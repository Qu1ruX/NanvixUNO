◆ superblock_empty()

PRIVATE struct superblock* superblock_empty ( void  )

Searches for a non-valid superblock entry in the superblock table.

Returns
If such superblock has been found, a pointer to it is returned. In this case, the superblock is ensured to be locked. However, if no such block has been found, a NULL pointer is returned instead.

◆ superblock_get()

PUBLIC struct superblock* superblock_get ( dev_t  dev)

Searches for a valid superblock in the superblock table which the device number equals the informed one.

Parameters
devDevice number.
Returns
If the requested superblock exists in the superblock table, a pointer to it is returned. In this case, the superblock is ensured to be locked. However, if no such superblock exists, a NULL pointer is returned instead.
Note
The device number should be valid.

◆ superblock_read()

PUBLIC struct superblock* superblock_read ( dev_t  dev)

Reads a superblock from the first block of a device. Once the read has completed, the magic number of the block is asserted and in-core fields are filled.

Parameters
devDevice number.
Returns
Upon successful completion, a pointer to the in-core superblock is returned. The superblock is ensured to be locked in this case. Upon failure, a NULL pointer is returned instead.
Note
The device number should be valid.
